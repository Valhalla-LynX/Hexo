[Log][PLG]Loki Microservices Log Collection

发表于2022-11-11|更新于2023-09-19|Log
|阅读量:

Intro

PLG is a label index based logging system, implemented same node server discovery for Kubernetes Pods.

Compared with ELK/PLG is more compatible with Kubernetes and has lower overhead because of label index.

Microservices deployment

On Kubernetes cluster using helm can be a easy way to install PLG service.

Deploy

Loki

For same Kubernetes cluster, a Loki service will be installed. And for each node in the cluster, Promtail installed as a component for Loki.

Loki is responsible for indexing and querying logs, and Promtail for locating the node pods or other services to transform the log to Loki node.

Grafana

For easy viewing of logs, Grafana is in need.

Whatever how the Grafana is deployed, just add a Loki data source.

View logs

Then you can view the logs by Grafana Explore.

In new version, there is a builder to construct query log method.
